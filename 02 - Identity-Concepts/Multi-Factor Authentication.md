
# Multi-Factor Authentication (MFA)

## Overview

Multi-Factor Authentication (MFA) is an authentication method that requires users to provide two or more forms of verification before access is granted.

MFA significantly reduces the risk of compromised credentials because an attacker must bypass multiple authentication factors instead of just a password.

---

## Authentication Factors

Authentication factors are commonly grouped into three categories.

### Something You Know

Information known by the user.

Examples:

- Password
- PIN
- Security questions

### Something You Have

A physical object possessed by the user.

Examples:

- Mobile phone
- Hardware token
- Authenticator application

### Something You Are

Unique biological characteristics.

Examples:

- Fingerprint
- Facial recognition
- Iris scan

Organizations typically combine two or more factors to verify identity.

---

## Common MFA Methods

### Authenticator Applications

Applications that generate verification codes or approve sign-in requests.

### Push Notifications

Users receive a notification on a trusted device and approve the authentication request.

### SMS Verification

A one-time code is sent to the user's phone.

### Voice Calls

An automated phone call verifies the user's identity.

### Hardware Tokens

Physical devices generate temporary authentication codes. 【1-5dc599】

## Conditional Access and MFA

Organizations can require MFA under specific conditions, such as:

- Sign-ins from unfamiliar locations.
- High-risk authentication attempts.
- Access from unmanaged devices.
- Administrative operations. 【1-5dc599】

---

## Benefits of MFA

- Stronger identity protection
- Reduced password-related attacks
- Improved account security
- Better compliance posture
- Lower risk of unauthorized access 【1-5dc599】

---

## Key Takeaways

- MFA requires multiple authentication factors.
- Passwords alone are no longer sufficient protection.
- MFA significantly reduces identity-based attacks.
- MFA is a core security recommendation in Zero Trust environments.
